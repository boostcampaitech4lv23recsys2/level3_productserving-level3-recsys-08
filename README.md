### 깃 Convention

1. **[Feat]**: 새로운 기능 추가, 코드 수정
2. **[Fix]**: 버그 수정
3. **[Docs]**: 문서 수정
4. **[Style]**: 코드 포맷팅, 세미 콜론 누락, 코드 변경 없는 경우
5. **[Refactor]**: 코드 리팩토링
6. **[Test]**: 테스트 코드, 리팩토링 테스트 코드 추가
7. **[Chore]**: 패키지 매니지, requirements 수정
